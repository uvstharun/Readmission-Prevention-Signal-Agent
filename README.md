# Readmission Prevention Signal Agent

An autonomous AI agent system that monitors newly discharged patients, scores 30-day readmission risk using a trained ML model, generates LLM-powered clinical narratives via Claude, and triggers evidence-based care transition workflows — all with a full audit trail and explainability layer.

This project demonstrates production-grade healthcare AI engineering: multi-agent orchestration, clinical ML, LLM integration, SHAP explainability, and a real-time monitoring dashboard.

## Setup

### 1. Clone and install dependencies
```bash
cd readmission-prevention-agent
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Configure environment
```bash
cp .env.example .env
# Edit .env and add your ANTHROPIC_API_KEY
```

### 3. Generate synthetic data
```bash
python -m src.data.synthetic_data_generator
```

### 4. Run feature engineering
```bash
python -m src.data.feature_engineering
```

### 5. Train the model
```bash
python -m src.models.model_trainer
```

### 6. Launch the API
```bash
uvicorn src.api.main:app --reload --port 8000
```

### 7. Launch the dashboard
```bash
streamlit run frontend/app.py
```

## Agent Descriptions

| Agent | Role |
|-------|------|
| **Orchestrator** | Coordinates all sub-agents in sequence for each discharge event |
| **PatientContextAgent** | Builds structured clinical context and identifies risk flags |
| **RiskScoringAgent** | Scores patient using XGBoost model + SHAP explainability |
| **CareGapAgent** | Evaluates 10 evidence-based care transition criteria |
| **ClinicalNarrator** | Calls Claude API to generate structured clinical narrative |
| **WorkflowTriggerAgent** | Triggers intervention bundle and adds patient to watchlist |
| **MonitoringAgent** | Monitors 30-day watchlist, detects signals, closes cases |

## Portfolio Story

**Clinical Problem:** 30-day readmissions cost the US healthcare system $26B annually. Most hospitals lack real-time, explainable tooling to identify high-risk patients at discharge and coordinate timely interventions.

**Agent Solution:** This system acts as an autonomous care transition coordinator — it scores every discharge in real time, explains *why* the patient is high risk using SHAP-driven clinical language generated by Claude, and triggers a personalized evidence-based intervention bundle before the patient leaves the building.

**Key Technical Decisions:**
- XGBoost chosen for primary model: best AUC-ROC on tabular clinical data, tree-based SHAP explanations are fast and stable
- Claude used for narrative generation: produces clinically credible, structured JSON outputs with care gap rationale
- SQLite for portability in portfolio demo; production would use PostgreSQL
- SMOTE applied to address the realistic 15% readmission base rate (class imbalance)
- SHAP TreeExplainer provides patient-level explanations in <100ms inference time
